## 1. 功能概述

### 1.1 功能定位

UI布局采用四栏结构：Dock导航栏 + 左侧边栏 + 主内容区 + 右侧抽屉面板，支持协作与分析同屏。

### 1.2 目标用户

- **主要用户**: 客服人员
- **次要用户**: 质检专员、管理层

### 1.3 核心价值

**对用户的价值**:
- 统一入口减少切换成本
- 右侧抽屉快速获取分析信息

**对业务的价值**:
- 统一交互与信息布局规范
- 降低新功能接入成本

### 1.4 优先级

- **优先级**: P0
- **RICE分数**: 86
  - Reach（影响用户数）: 150+用户
  - Impact（对用户价值）: 高
  - Confidence（成功把握）: 85%
  - Effort（开发成本）: 2周

## 2. 功能需求

### 2.1 用户故事

**US-01**: 作为客服，我希望在同屏查看对话与分析面板，以便快速处理问题。

**US-02**: 作为质检专员，我希望切换质检面板而不离开当前任务视图。

**US-03**: 作为管理层，我希望在不同设备下保持一致的导航与布局体验，以便快速定位关键信息。

### 2.2 功能列表

| 功能项 | 描述 | 优先级 | 依赖 | 工作量 |
|-------|------|--------|------|--------|
| Dock导航 | 模块切换入口 | P0 | 无 | 2天 |
| 左侧边栏 | 列表与子菜单 | P0 | 模块数据 | 2天 |
| 主内容区 | 内容渲染区域 | P0 | 模块组件 | 2天 |
| 右侧抽屉 | 分析/AI/质检 | P0 | 抽屉组件 | 3天 |
| 宽度拖拽 | 左右侧宽度调整 | P1 | 布局容器 | 2天 |
| 布局配置 | 默认宽度与模式 | P1 | 配置服务 | 2天 |
| 布局持久化 | 用户级保存与恢复 | P1 | 用户配置服务 | 2天 |
| 布局可访问性 | 键盘与无障碍支持 | P1 | 前端基础组件 | 1天 |

### 2.3 详细规格说明

#### 功能1：Dock导航

**功能描述**: 提供消息、知识、工具、报表、权限等模块入口。

**外部依赖与降级展示**:
- **导航配置服务**: 若加载失败，使用内置默认导航。

**交互行为**:
1. 点击Dock项 → 切换主内容区
2. Dock顺序中“权限”位于“报表”下方

**边界条件**:
- 当前模块未完成加载 → 显示加载状态

**权限要求**: 详见[权限与入口管理说明](2.10-Permission-Management-PRD.md)

---

#### 功能2：左侧边栏

**功能描述**: 展示当前模块的列表或子菜单，支持折叠与滚动。

**外部依赖与降级展示**:
- **模块数据服务**: 若不可用，显示“加载失败”，并提供重试。

**交互行为**:
1. 点击列表项 → 主内容区更新
2. 点击折叠 → 左侧栏缩小

**边界条件**:
- 无列表项 → 显示空状态
- 折叠后 → 仅显示图标与提示

---

#### 功能3：主内容区

**功能描述**: 承载模块主视图，支持面包屑与页内滚动。

**交互行为**:
1. 路由切换 → 主内容区重渲染
2. 主内容区滚动 → 右侧抽屉保持固定

**边界条件**:
- 内容加载慢 → 显示骨架屏

---

#### 功能3：右侧抽屉

**功能描述**: 承载经典分析、AI辅助、质检等面板。

**外部依赖与降级展示**:
- **抽屉内容服务**: 若不可用，显示“内容加载失败”。

**交互行为**:
1. 点击入口 → 抽屉展开
2. Tab切换 → 内容切换

**边界条件**:
- 非对话模块 → 抽屉默认关闭

---

#### 功能4：宽度拖拽

**功能描述**: 支持左侧边栏与右侧抽屉的宽度拖拽调整。

**交互行为**:
1. 拖拽分隔线 → 实时调整宽度
2. 松开拖拽 → 保存当前宽度

**边界条件**:
- 小于最小宽度 → 限制并回弹

---

#### 功能5：布局配置

**功能描述**: 支持读取默认宽度与抽屉模式配置。

**外部依赖与降级展示**:
- **配置服务**: 若不可用，使用内置默认值（左侧280px，抽屉360px）。

**交互行为**:
1. 进入工作台 → 加载配置并应用

---

#### 功能6：布局持久化

**功能描述**: 支持用户级保存布局偏好（左右侧宽度、抽屉状态）。

**外部依赖与降级展示**:
- **用户配置服务**: 若不可用，使用本地存储兜底。

**交互行为**:
1. 宽度调整/抽屉切换 → 本地与服务端保存
2. 下次登录 → 读取并恢复

**边界条件**:
- 旧版本配置不兼容 → 使用默认值

---

#### 功能7：布局可访问性

**功能描述**: 提供键盘导航与读屏支持。

**交互行为**:
1. Tab顺序 → Dock → 左侧栏 → 主内容 → 右侧抽屉
2. 快捷键切换抽屉 → `Alt + D`

**边界条件**:
- 快捷键冲突 → 提示用户并允许关闭

## 3. UI设计

### 3.1 页面布局

#### 布局方案1：四栏结构

**页面结构**:
```
┌────────────────────────────────────────────────────────┐
│ Header (56px)                                           │
├──────┬──────────────────┬─────────────────┬───────────┤
│ Dock │ 左侧边栏 (可拖拽) │ 主内容区        │ 右侧抽屉  │
└──────┴──────────────────┴─────────────────┴───────────┘
```

**布局说明**:
- Dock宽度固定64px
- 左侧边栏默认280px，可拖拽
- 右侧抽屉默认360px，可收起
- 主内容区最小宽度720px

### 3.2 组件说明

#### 组件1：Dock导航条

**组件类型**: 纵向导航

**位置**: 左侧固定

**交互行为**:
- 点击切换模块

---

### 3.3 样式规范

- 颜色、字体、间距与全局一致

### 3.4 模块功能布局说明

为确保布局规范真实反映项目使用场景，以下按模块给出四栏结构中的功能分配与主要内容：

| 模块 | Dock入口 | 左侧边栏 | 主内容区 | 右侧抽屉 |
|------|---------|---------|---------|---------|
| 对话工作台 | 对话 | 会话列表、分组、筛选 | 会话窗口、消息流、客户信息摘要 | 质检面板、AI建议、会话统计 |
| 质检中心 | 质检 | 任务列表、抽检规则 | 质检详情、问题列表 | 质检评分、模型解释 |
| 报表中心 | 报表 | 报表目录、时间筛选 | 报表图表与表格 | 指标说明、导出配置 |
| 知识库 | 知识 | 文档目录、标签筛选 | 文档详情、编辑器 | 推荐关联、引用记录 |
| 工单 | 工单 | 工单列表、状态筛选 | 工单详情与处理流 | 处理建议、历史关联 |
| 权限管理 | 权限 | 角色/用户/策略列表 | 权限配置与策略详情 | 变更记录、风险提示 |

**布局统一规则**:
- Dock仅放一级模块入口，禁止放二级功能
- 左侧边栏承载导航与筛选，不放重交互表单
- 主内容区为业务主流程，优先保留可视高度
- 右侧抽屉仅承载辅助信息与快速操作

**二级页面布局细化**:

1) 对话工作台
- 会话列表页: 左侧边栏为会话分组/标签筛选，主内容区为会话预览列表，右侧抽屉为当前会话概要与质检提示
- 会话详情页: 左侧边栏为会话列表，主内容区为消息流与输入区，右侧抽屉为客户画像/AI建议/质检评分
- 客户画像页: 左侧边栏为客户列表，主内容区为客户画像与历史工单，右侧抽屉为关联风险与推荐动作

2) 质检中心
- 任务列表页: 左侧边栏为任务筛选与规则组，主内容区为任务列表，右侧抽屉为抽检策略与进度
- 质检详情页: 左侧边栏为问题列表，主内容区为质检详情与证据，右侧抽屉为评分维度与模型解释
- 规则配置页: 左侧边栏为规则目录，主内容区为规则编辑器，右侧抽屉为影响范围与版本说明

3) 报表中心
- 报表总览页: 左侧边栏为报表目录，主内容区为关键指标卡片与趋势图，右侧抽屉为指标释义与口径
- 报表详情页: 左侧边栏为筛选条件，主内容区为图表/表格，右侧抽屉为导出配置与订阅设置

4) 知识库
- 文档列表页: 左侧边栏为目录/标签，主内容区为文档列表与预览，右侧抽屉为推荐与引用统计
- 文档详情页: 左侧边栏为目录树，主内容区为文档详情与编辑，右侧抽屉为版本记录与引用关系

5) 工单
- 工单列表页: 左侧边栏为状态/优先级筛选，主内容区为工单列表，右侧抽屉为处理建议与SLA提示
- 工单详情页: 左侧边栏为工单流转记录，主内容区为工单详情与处理操作，右侧抽屉为关联会话与风险提示

6) 权限管理
- 角色列表页: 左侧边栏为角色与分组，主内容区为角色列表与详情，右侧抽屉为变更记录与风险提示
- 用户权限页: 左侧边栏为用户列表，主内容区为权限配置，右侧抽屉为权限影响范围与审计摘要

### 3.4 响应式设计

| 断点 | 宽度范围 | 布局调整 |
|-----|---------|---------|
| Desktop | >1200px | 四栏布局 |
| Tablet | 768-1199px | 抽屉覆盖层 |
| Mobile | <768px | Dock折叠为底部导航 |

- **恢复方式**: 刷新

## 5. 接口定义

### 5.1 获取布局配置

**接口路径**: `/api/layout/config`

**请求方法**: GET

**响应体**:
```json
{
  "leftSidebarWidth": 280,
  "drawerWidth": 360,
  "drawerMode": "analysis",
  "dockPinned": true
}
```

---

### 5.2 保存布局配置

**接口路径**: `/api/layout/config`

**请求方法**: POST

**请求体**:
```json
{
  "leftSidebarWidth": 300,
  "drawerWidth": 380,
  "drawerMode": "quality",
  "dockPinned": true
}
```
