# 性能测试方案

> **文档版本**: v1.0
> **更新日期**: 2026-01-27

---

## 1. 目标指标

- API P95 < 500ms（核心接口）
- 并发用户 500+（阶段性）
- 消息吞吐 500+ msg/min

---

## 2. 工具建议

- k6 / Locust / JMeter（三选一）

---

## 3. 核心压测场景

- 登录与鉴权
- 会话列表/消息查询
- 需求创建/状态更新
- 任务创建/完成

---

## 4. 数据准备

使用 `docs/testing/TEST_DATA.md` 中的数据集准备脚本或 mock 数据。

---

**维护团队**: QA / 后端
